# Functional Status

_Last updated: 2026-04-12_

This document captures where the current alpha build stands versus the roadmap described in `README.md` and `ARCHITECTURE.md`. Use it to see what already works, what is partially there, and which gaps remain before we can call the heap analyzer "functionally complete".

## Snapshot
- ✅ **v0.2.0 is the current release** and includes a documentation-aligned core crate restructure from flat files into grouped module directories under `core/src/` (`hprof/`, `graph/`, `analysis/`, `mapper/`, `report/`, `fix/`, `mcp/`).
- ✅ **M1.5 real-world hardening landed**: `HEAP_DUMP_SEGMENT` parsing now uses the correct HPROF tag constant (`0x1C`), record-tag naming matches the spec, and the workspace now validates `parse`, `analyze`, `leaks`, and `gc-path` against a real JVM heap dump when the optional `resources/test-fixtures/heap.hprof` fixture is present.
- ✅ **HPROF parser** streams headers/record stats and produces a record-category histogram + heap summary without loading entire dumps into RAM.
- ✅ **Object-graph pipeline** now lives under `core::hprof/`, where the shared object graph, binary HPROF parser, and test fixtures are grouped together for the parsing domain. `ObjectGraph` can retain raw instance `field_data`, selective `byte[]`/`char[]` contents, and parsed `STACK_TRACE` / `STACK_FRAME` records for higher-level analyzers, but that retention is now opt-in through `ParseOptions { retain_field_data: true }` so default `analyze`, `leaks`, and `gc-path` runs stay on the lean path.
- ✅ **M3 foundation tooling landed**: `core::hprof::tags` now centralizes top-level and heap-dump sub-record constants for the parser, binary parser, fixtures, and GC-path traversal, and the repository now includes Criterion bench targets plus `scripts/measure_rss.sh` for throughput/RSS baselining.
- ✅ **M3-P1-B2 core analysis features landed**: `analyze_heap()` now emits grouped histograms, unreachable-object summaries, and MAT-style suspect scoring; `diff_heaps()` now attaches class-level deltas when both dumps build object graphs; and the CLI exposes `analyze --group-by class|package|classloader` plus class-diff table output.
- ✅ **M3 Phase 2 investigation features landed**: `analyze_heap()` can now attach optional thread, string, collection, and top-instances reports; the core ships `inspect_threads()`, `analyze_strings()`, `inspect_collections()`, and `find_top_instances()`; and the CLI exposes `analyze --threads --strings --collections --top-instances` plus tuning flags `--top-n` and `--min-capacity`.
- ✅ **Step 11 large-dump scaling validation landed**: `ParseOptions` now controls field-data retention, `analyze_heap()` only opts into higher-memory parsing when string/collection/thread investigation is requested, `detect_leaks()` and `gc-path` stay lean by default, the enhanced `scripts/measure_rss.sh` profiles `parse`, `analyze`, and `leaks` with ratio markers and `/proc/PID/status` fallback sampling, and the dense synthetic validation batch now clears ~500 MB / ~1 GB / ~2 GB tiers with default-path RSS ratios around 2.87x-2.90x and investigation-path ratios around 3.89x-3.92x.
- ✅ **CLI surface** (`parse`, `leaks`, `analyze`, `diff`, `map`, `gc-path`, `query`, `explain`, `chat`, `fix`, `serve`, `config`) all call into the shared core. `parse` summary sections and `leaks` output now render aligned terminal tables at the CLI boundary, `chat` now ships as a CLI-first leak-focused conversation mode with `/focus <leak-id>`, `/list`, `/help`, and `/exit`, and follow-up disclosure sections still appear when width-bounded cells truncate record-category names, leak IDs, or class names.
- ✅ **Zero-result UX** is now explicit: `mnemosyne leaks` prints `No leak suspects detected.` when analysis completes without candidates, and `mnemosyne fix` already returns `No fix suggestions available for the provided criteria.` when nothing matches the requested filters.
- ✅ **Error handling** now uses structured `CoreError` variants plus CLI `hint:` lines for missing heap dumps, wrong file types, HPROF header parse failures, and invalid config files.
- ✅ **Leak detection** is now unified: `detect_leaks()` attempts object-graph → dominator → retained-size analysis first, then falls back to heuristics with explicit `ProvenanceKind::Fallback` markers. `explain --leak-id`, `fix --leak-id`, and MCP `explain_leak` now validate the requested leak identifier up front and return `CoreError::InvalidInput` for unknown IDs instead of silently widening to the full leak set.
- ✅ **Graph/dominator view** now supports real retained sizes in both `analyze_heap()` and `detect_leaks()`, with the lightweight summary preview retained only as fallback.
- ✅ **GC path finder** now prefers full `ObjectGraph` BFS first, then falls back to a budget-limited `GcGraph`, then synthetic paths when the heap lacks enough detail.
- ✅ **Object-graph navigation API** now exposes `get_object(id)`, `get_references(id)`, and `get_referrers(id)` for programmatic heap exploration.
- ✅ **M3 Phase 3 parity features landed**: `analyze_heap()` now attaches optional classloader reports, shared report renderers include classloader sections, the CLI exposes `analyze --classloaders`, the core ships a minimal OQL parser + executor for built-in fields, the CLI exposes `query`, the MCP server exposes `analyze_heap` and `query_heap`, and `analyze --profile overview|incident-response|ci-regression` now applies preconfigured investigation defaults.
- ⚠️ **AI insights** now support three explicit modes: `rules` (default), `stub`, and `provider`. Provider mode now supports YAML-backed prompt templates through an embedded default plus optional `[ai.prompts].template_dir` overrides, plus Step `14(d)` privacy controls under `[ai.privacy]` for `heap_path` redaction, regex-based prompt redaction, opt-in hashed audit logging before external model calls, and a minimal prompt-budget guard. Step `14(e)` now ships a partial CLI-first conversation slice through `mnemosyne-cli chat <heap.hprof>` with bounded in-process history and top-3 shortlist context, and `fix` / `propose_fix` now attempt an AI-backed source-aware patch path when provider mode plus mapped source context are available. MCP/session semantics still remain pending.
- ✅ **Report/export** supports Text/Markdown/HTML/TOON/JSON with `--output-file`. HTML output is XSS-hardened; TOON values are properly escaped. Provenance markers are rendered in all non-JSON formats.
- ✅ **Provenance system** labels synthetic, partial, fallback, and placeholder data across analysis responses, leak insights, GC paths, and fix suggestions. CLI and report renderers surface these markers to consumers.
- ✅ **Output hardening** — HTML escaping prevents XSS in report output; TOON escaping handles control characters and backslashes correctly.
- ✅ **Development workflow** now includes GitHub Actions CI, the `test-fixtures` cargo feature for canonical fixture reuse, real-world HPROF validation that skips gracefully when the large fixture is absent, and 194 passing tests.
- ✅ **Release automation** now validates `v*` tags against the workspace version, cross-compiles `mnemosyne-cli` for five targets, packages archives, publishes GitHub Releases with attached binaries, and builds/pushes the GHCR Docker image on tagged releases.
- ✅ **Packaging metadata + Homebrew scaffolding** now cover crates.io-ready workspace metadata across `Cargo.toml`, `core/Cargo.toml`, and `cli/Cargo.toml`, plus a macOS Homebrew formula for both Apple Silicon and Intel release archives.
